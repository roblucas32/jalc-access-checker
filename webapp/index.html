Accessibility Checker

Drop in a caption file, a Canvas page export, or a Word document. Get an instant WCAG 2.1 A/AA structural report — no upload, no waiting.

🔒 Files never leave your browser. Nothing is uploaded or stored.

Checks .vtt and .srt caption files against JALC's captioning standards: lines that are too long, timing that overlaps or is too short, captions with missing text, and pacing that's too fast to read.

Based on the DCMP Captioning Key (32 chars/line, ~1.33s minimum duration, reading-rate targets) and BBC Subtitle Guidelines, which WCAG 2.1's own captioning criterion (Success Criterion 1.2.2) points to for implementation specifics. Defaults are adjustable below.

Checks a Canvas page or an exported .html file against WCAG 2.1 A/AA structural criteria: alt text, headings, contrast, forms, links, and tables.

WCAG 2.1 AA is the actual federal standard here, not just a best practice — the DOJ's ADA Title II final rule requires public colleges' web content to meet WCAG 2.1 Level AA by April 2027/2028 depending on jurisdiction size.

Note: when checking a Canvas page, "missing main heading" is intentionally not flagged — Canvas reserves Heading 1 for the page title (a separate field the Rich Content Editor doesn't let you set on the body), so a well-built page correctly starts its content at Heading 2. This exception doesn't apply to uploaded .html files, which do need their own <h1>.

        Checks every page, Word document, and Studio video in a course at once, and produces one combined report — instead of checking items one at a time.

        Only checks content visible to students by default — unpublished pages and hidden/locked files are skipped in the main scan and listed separately, since they aren't real accessibility issues yet (students can't see them). You can run a second scan of just that draft content afterward if you want to get ahead of it before publishing.

        Videos are checked regardless of publish status — Canvas Studio doesn't expose a simple "published" flag for videos the way it does for pages and files, so there's no reliable way to separate draft from live videos here. If a flagged video is intentionally unpublished, that's worth knowing when you read the report.

        Large courses are capped at 150 items to keep the scan from running indefinitely — if a course has more than that, you'll see a warning and the scan covers the first batch. Scan a narrower selection (e.g. one module's content) if a course regularly exceeds this.
